### ❌ 沒做（附理由）

- ❌ **存進 Supabase / 任何資料庫**。Spec 在 system requirements 寫「需要永久儲存狀態」，沒明確指定資料庫。Brainstorming 階段使用者要求「越輕量越好」，收斂到 localStorage（zero infra；換裝置不同步是接受的 trade-off）。
- ❌ **抽籤動畫**（fade in / 翻牌等）。Spec line 194 標 open question，預設「無，先做 functional」。實作後仍維持「無動畫」，只有 button `active:scale-[0.98]` 觸覺反饋 + `toast.success` 提示。

### ➕ 多做了（附理由）

實作中加入、spec 沒指定的東西：

- ➕ **`isBrowser()` SSR guard** in `lib/nocturne.js`。Next.js App Router 會 SSR 一次，`window`/`localStorage` 在 server 端不存在；helpers 全部包了 guard 才不會 hydration error。
- ➕ **`mounted` gate + lazy initializer 模式** in page.jsx。React 19 + React Compiler 的 `react-hooks/set-state-in-effect` lint rule 禁止「直接在 useEffect 內 setState」，所以 state init 改用 `useState(() => typeof window === 'undefined' ? DEFAULT : load())`，再用 mounted gate 避免 hydration mismatch。Spec 沒指定這個 pattern。
- ➕ **`recordDraw` 的 cutoff anchor**：把 30 天 retention 的計算錨點從 `now` 改成 `tonightDate(now)` 算出的日期，讓 6pm 呼叫和 8pm 呼叫得到一致的 cutoff。code-quality reviewer 提的 SHOULD-fix。
- ➕ **`recordDraw` 的 future-dated guard**：filter 從 `d.night_date !== nightDate` 改成 `d.night_date < nightDate`，順手剔除任何系統時鐘被往前撥造成的 future entries。code-quality reviewer 提的 SHOULD-fix。
- ➕ **`saveWeights` 與 `recordDraw` 的 try/catch wrappers**。`localStorage.setItem` 在 quota exhausted / Safari private mode 會 throw，原本沒包；codex review 提了 P2，採納。
- ➕ **`THEMES_BY_ID` lookup map** in `configs/nocturne-themes.js`：`Object.fromEntries(THEMES.map(...))`。spec 只說放在 config，沒指定要不要 derived map；page 寫起來方便所以加。
- ➕ **`lastNNightDates(n, now)` helper**：HistoryStrip 計算 7 個日期格用，集中在 `lib/nocturne.js` 跟 `tonightDate` 同檔。Spec 沒列這個 helper。
- ➕ **主題 id 用英文 slug**（`dev`/`study`/`play`/`solo`/`free`）。Spec 沒明確要求；設計階段為了「穩定 identifier，不依賴中文字串」決定。

## 3. 程式碼片段

### `lib/nocturne.js` — constants + `tonightDate` + `drawTheme`

```javascript
const STORAGE_KEY_WEIGHTS = 'nocturne:weights';
const STORAGE_KEY_DRAWS = 'nocturne:draws';
const RETENTION_DAYS = 30;

export const THEME_IDS = ['dev', 'study', 'play', 'solo', 'free'];
export const DEFAULT_WEIGHTS = { dev: 25, study: 20, play: 20, solo: 20, free: 15 };

export function tonightDate(now = new Date()) {
    if (now.getHours() >= 19) {
        return toLocalDateString(now);
    }
    const yesterday = new Date(now);
    yesterday.setDate(yesterday.getDate() - 1);
    return toLocalDateString(yesterday);
}

export function drawTheme(weights, rng = Math.random) {
    const entries = THEME_IDS.map((id) => {
        const raw = Number(weights?.[id] ?? 0);
        return [id, Number.isFinite(raw) && raw > 0 ? raw : 0];
    });
    const sum = entries.reduce((acc, [, w]) => acc + w, 0);
    if (sum === 0) {
        return THEME_IDS[Math.floor(rng() * THEME_IDS.length)];
    }
    const r = rng() * sum;
    let cumulative = 0;
    for (const [id, w] of entries) {
        cumulative += w;
        if (r < cumulative) return id;
    }
    return entries[entries.length - 1][0];
}
```

### `lib/nocturne.js` — `recordDraw`（含 try/catch + cutoff anchor + future-dated guard）

```javascript
export function recordDraw(themeId, now = new Date()) {
    if (!THEME_IDS.includes(themeId)) {
        throw new Error(`Unknown theme id: ${themeId}`);
    }
    const nightDate = tonightDate(now);
    const cutoffBase = new Date(`${nightDate}T12:00:00`);
    cutoffBase.setDate(cutoffBase.getDate() - RETENTION_DAYS);
    const cutoffString = toLocalDateString(cutoffBase);

    const existing = loadDraws();
    const pruned = existing.filter((d) => d.night_date >= cutoffString && d.night_date < nightDate);
    const next = [{ night_date: nightDate, theme: themeId, drawn_at: now.toISOString() }, ...pruned].sort((a, b) =>
        a.night_date < b.night_date ? 1 : -1,
    );

    if (isBrowser()) {
        try {
            window.localStorage.setItem(STORAGE_KEY_DRAWS, JSON.stringify(next));
        } catch {
            // storage write failed — return the new array but skip persistence
        }
    }
    return next;
}
```

### `app/nocturne/page.jsx` — state init pattern（lazy + mounted gate + init wrapper）

```jsx
const [mounted, setMounted] = useState(false);
const [now] = useState(() => new Date());
const [weights, setWeights] = useState(() => (typeof window === 'undefined' ? DEFAULT_WEIGHTS : loadWeights()));
```

## 4. 技術選型紀錄

每項只記「考慮過什麼 / 選了什麼 / 為什麼選這個」，不評價。

### 持久化方案

- **A**：Supabase 兩張 table（5-row weights + draws table）
- **B**：Supabase 1 張 table（jsonb weights + draws table）
- **C**：weights localStorage + draws Supabase（hybrid）
- **D**：完全 localStorage（zero infra）

選 **D**。理由：使用者明確指示「越輕量越好」、5 個主題是 hard-coded 永不增減、spec 接受換裝置不同步、philosophy 對齊「抽完就走」。

### 抽籤演算法

- **A**：簡單 cumulative weight（sum + binary walk）
- 沒考慮其他方案（A 已足夠 5 元素 vector）

選 **A**。Sum=0 加 defensive uniform fallback。`rng` 參數可注入（為了未來除錯方便，沒寫測試但保留 inject point）。

### 7pm 換日邏輯位置

- **A**：純 client 計算（`now.getHours() >= 19`）
- **B**：抓 server time
- **C**：用 SQL 算

選 **A**。理由：feature 走 D 路線（無 server）後，B/C 不適用。即使有 server，spec 接受時鐘漂移不處理，A 已足夠。

### UI 版型

- **A**：垂直捲動，主視覺 → 歷史 → 設定（折疊）
- **B**：底部 tab bar 切換 view
- **C**：主視覺全螢幕 + 浮動 icon 開 modal/drawer

選 **A**。理由：spec line 95「同一個 app 介面內**流動**」、極簡哲學、`RecordPageLayout` 已是 vertical flex，直接 fit。

### 主題文案資料源

- **A**：寫進 `configs/nocturne-themes.js`
- **B**：存 Supabase

選 **A**。理由：5 主題 hard-coded、文案是 spec 一部分（不該 runtime mutable）、走 D 路線後 B 不適用。

### 自由之夜視覺

- **A**：同卡片外框，但內容只渲染主題名 + 一句話（隱藏空欄位）
- **B**：完全不同視覺，無框、hero text + 宣告（poster style）

選 **B**。理由：spec line 38 AC 寫「**不套用其他四主題的卡片格式**，獨立呈現」、「單行宣告」這個用詞、philosophy 「掙來的自由」需要視覺對比。

### State init pattern

- **A**（spec 原版）：`useState(DEFAULT)` + `useEffect` 內 setState 從 localStorage 載入 + mounted gate
- **B**：lazy `useState(() => load())` + mounted gate + `init()` wrapper 在 `useEffect`

選 **B**。理由：A 觸發 React Compiler `react-hooks/set-state-in-effect` lint rule。B 是 implementer subagent 自主 deviation；行為等價（mounted gate 控制首次 render）。
