# 一句話定義

> 一個每晚 7 點到家後抽出今晚主題的個人儀式器，讓 5 小時自由時間有方向感而不潰堤。

# Project Vibe & Goal

這個專案的靈魂是「保護儀式感」。

LeetCode 寫完後的 5 小時自由時間最容易潰堤——攤在沙發上滑手機到睡前、醒來懊悔。夜籤把「決定今晚做什麼」這個動作從「攤著合理化」前移到「打開 app 抽一張」，用儀式中斷潰堤路徑。

工具不追蹤、不打勾、不獎勵——抽完就走，剩下整晚交給卡片上的入場動作和主場規則。靈魂是極簡的：一張卡、一個動作、一晚沉浸。

# Core Workflow

- 7pm 為換日界線 → 觸發「可抽籤」狀態
- 使用者打開 app → 一天一抽 → 按權重隨機產出主題卡
- 卡片顯示：主題名、入場動作（最醒目）、主場規則、5 小時節奏建議
- 已抽過則直接顯示當晚卡片，不能重抽
- 隔天 7pm 才解鎖下一次抽籤
- 歷史頁顯示最近 7 晚的主題分佈（沒抽的晚上呈現空白格）

# Features (P0 / MVP)

- [使用者打開 app 看到當晚的主題卡片] — AC: 已抽過顯示卡片內容；未抽過顯示「抽籤」按鈕
- [按權重隨機抽出主題] — AC: 五個主題權重決定機率，總和不必為 100，依相對比例計算
- [一天只能抽一次] — AC: 抽完當下到下一個 7pm 之間，「抽籤」按鈕停用或不出現
- [7pm 為換日界線] — AC: 系統時間進入 7pm 後視為「新的一晚」，抽籤解鎖
- [卡片完整資訊呈現] — AC: 主題名、入場動作（獨立區塊、視覺最醒目）、主場規則、節奏建議（如該主題有的話）
- [自由之夜獨立視覺] — AC: 不套用其他四主題的卡片格式，獨立呈現（例如單行宣告，不顯示「入場動作：無」這種空欄位）
- [權重設定可調可保存] — AC: 五個主題各自有權重值（預設 30/20/20/15/15），可調整並持久保留
- [歷史紀錄] — AC: 顯示最近 7 晚的「日期 + 主題」分佈；沒抽的晚上以空白格呈現；資料保留最近 30 天

**五個主題的卡片內容（實作時直接照抄）**

1. **開發之夜**（預設權重 30）
    - 入場動作：打開 IDE，至少寫一個 commit（非 LeetCode，WIP / docs / refactor 都算）
    - 主場規則：整晚螢幕主要時間都在 IDE / 終端機 / 專案相關的 docs。可以查資料但不開遊戲、不看動畫、不滑社群。
    - 節奏建議：暖機段 30-60 分鐘（找回上次脈絡、修小 bug、讀自己上次的 commit）/ 主攻段 2-3 小時（今晚的核心進度）/ 收尾段 30-60 分鐘（整理、寫 commit message、列明天的 TODO）/ 中間至少站起來一次

2. **學習之夜**（預設權重 20）
    - 入場動作：選定一個來源（日文 / 技術文章 / 書），計時器設 50 分鐘，按下開始
    - 主場規則：整晚圍繞幾個學習主題，形式可以分段流動。可以切換到別的學習主題（例如日文 + 技術混搭）。
    - 節奏建議：主動學習段（番茄鐘 50 分鐘 × 2-3 輪，中間休息 10 分鐘不滑手機）/ 消化段（看相關影片、做筆記、整理今晚學到的）/ 延伸段（聽 podcast、看日劇、讀延伸文章）/ 例「日文夜」：Anki + 文法書 + 休息 + 有字幕的日劇 + 整理生字

3. **娛樂之夜**（預設權重 20）
    - 入場動作：打開 PS5 進入遊戲，或開始播放動畫的第一集
    - 主場規則：整晚就是玩 / 看，不一邊玩一邊滑手機、不開副螢幕做別的事。中途換遊戲、換動畫、換到看實況都行，但必須在「娛樂內容」這個範圍裡沉浸。
    - 節奏建議：注意娛樂疲勞，連續玩 5 小時動作遊戲第 4 小時會麻木。可以中途換形式保持新鮮感（遊戲 → 動畫 → 不同類型遊戲）。

4. **獨處之夜**（預設權重 15）
    - 入場動作：手機調靜音放到視線外，打開 AI 對話框，輸入第一句：「今天我想聊***」或「最近我在想***」
    - 主場規則：整晚在「跟自己對話」的範圍裡流動，不開遊戲、不看影片娛樂內容、不滑社群。可以：跟 AI 深入聊（建議至少 60-90 分鐘）/ 寫東西（日誌、長文、給未來自己的信、整理某個想法）/ 散步思考（室內室外都行）/ 安靜做手作（泡茶、整理書、手寫）/ 放音樂發呆，讓思緒漫遊。共通點：手機不參與、沒有外部刺激源、跟自己有空間。中途想到工作的事，記在另一個地方明天再說。

5. **自由之夜**（預設權重 15）
    - 入場動作：無
    - 主場規則：滑手機、看 YouTube、發呆、加班、睡覺都行——這是掙來的自由。

**MVP 破關條件 (Exit Condition)**：

> 在 7pm 後第一次打開 app，看到「抽籤」按鈕。按下後畫面切換成主題卡片，入場動作獨立區塊以最醒目的視覺呈現，主場規則與節奏建議在下方顯示。同一晚再次打開 app，直接看到剛才那張卡片，沒有「抽籤」按鈕（或按鈕停用）。歷史頁可以看到包含今晚在內的最近 7 晚紀錄，每一晚顯示日期 + 主題或空白格。隔天 7pm 後再打開 app，「抽籤」按鈕重新出現可按。自由之夜抽到時，結果頁不顯示「入場動作：無」這種空欄位，而是有獨立的呈現方式。

# Features (P1)

- 權重調整 UI（滑桿 / 數字輸入 / 點擊增減皆可，看一頁式空間決定）
- 歷史紀錄頁的視覺呈現（如何在小空間內讓 7 天分佈一目了然，包含空白格的訊號感）

# 顯式排除 (Out of Scope)

故意不做、實作時不要偷偷加進來：

- **任何完成 / 打勾 / 進度追蹤功能**——這個工具是「抽完就走」的儀式器，不是行為追蹤器
- **重抽機制**——一天就是一次，重抽會破壞儀式承諾感
- **連續同主題的提醒 / 慶祝 / 跳出對話框**——權重已經處理長期分配，不需要短期干預
- **通知提醒 / streak / 成就系統**——會把工具從儀式滑進遊戲化，違背原意
- **跨主題切換偵測**——主場規則靠使用者自律守護，不靠系統監控
- **帳號系統**——單人工具
- **匯出 / 統計分析報表**——歷史頁能看 7 天 / 留 30 天就夠
- **權重 0 = 暫時關閉某主題的特殊處理**——暫不處理此語意，使用者自行避免歸 0

# System Requirements & Constraints

- 需要永久儲存狀態（當晚抽到的主題、權重設定、歷史紀錄 30 天）
- 資料量極小（每天一筆紀錄、五個權重值、歷史保留 30 天上限）
- 對延遲沒有要求（單機操作）
- 需要可靠地判斷「7pm 換日」邊界——以系統時間為準
- 一頁式 UI（抽籤、結果、設定、歷史在同一個 app 介面內流動）
- 預期實作平台：個人手機 app

# Known Edge Cases

- **7pm 邊界臨界**：6:59 抽到一張、7:01 理論上又能抽——確認接受此邊界視為 feature，不額外加「同自然日 + 跨 7pm」的雙條件鎖
- **沒抽的晚上**：歷史紀錄以空白格呈現，作為「那晚沒儀式」的唯一訊號（這是工具裡僅存的訊號設計）
- **權重全為 0**：邏輯上無法抽籤，預設權重避免此狀態，使用者自行避免歸零
- **時區 / 系統時間被改**：不處理，假設使用者在固定時區使用、不會手動調系統時間
- **app 在 6:59 開啟、7:00 自然跨時**：使用者剛抽完看到結果，不會即時刷新按鈕狀態，這個邊界忽略（下次打開 app 才更新）
- **自由之夜結果頁的格式異質**：必須獨立 UI 處理，不能套用其他主題的「入場動作 / 主場規則」格式

# 開放問題 (Open Questions)

- 權重調整的 UI 形式（滑桿 vs 數字輸入 vs 點擊增減）——實作時試過後決定
- 歷史紀錄空白格的視覺強度（淡灰、留白、emoji 標示）——試做後再調
- 主場規則 / 節奏建議的展開折疊——預設全展開還是可折疊，視卡片高度決定
- 五個主題的視覺色調 / icon 設計——實作時可以加，但不在規格內
