# Nocturne 夜籤 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build the `/nocturne` page — a once-per-night ritual draw that picks one of five hard-coded themes by adjustable weights, persists state in `localStorage`, and shows a 7-night history strip and collapsible weight settings.

**Architecture:** Pure-client feature, zero server. State lives in `localStorage` under the `nocturne:` namespace. Page wraps the project's existing `RecordPageLayout`. All sub-components live inline in `app/nocturne/page.jsx`. Pure logic (date math, weighted draw, IO) is extracted into `lib/nocturne.js` so the page stays a thin shell over data.

**Tech Stack:** Next.js 16 App Router, React 19, JavaScript (no TypeScript), Tailwind CSS 4, lucide-react icons, sonner toasts. Existing `RecordPageLayout` provides header + Toaster + page background.

**Spec:** `docs/specs/2026-05-01-nocturne-design.md` (architecture + decisions). Source-of-truth for content / edge cases: `nocturne.md`.

```javascript
const STORAGE_KEY_WEIGHTS = 'nocturne:weights';
const STORAGE_KEY_DRAWS = 'nocturne:draws';
const RETENTION_DAYS = 30;

export const THEME_IDS = ['dev', 'study', 'play', 'solo', 'free'];
export const DEFAULT_WEIGHTS = { dev: 30, study: 20, play: 20, solo: 15, free: 15 };

function isBrowser() {
    return typeof window !== 'undefined' && typeof window.localStorage !== 'undefined';
}

function pad2(n) {
    return String(n).padStart(2, '0');
}

function toLocalDateString(d) {
    return `${d.getFullYear()}-${pad2(d.getMonth() + 1)}-${pad2(d.getDate())}`;
}

export function tonightDate(now = new Date()) {
    if (now.getHours() >= 19) {
        return toLocalDateString(now);
    }
    const yesterday = new Date(now);
    yesterday.setDate(yesterday.getDate() - 1);
    return toLocalDateString(yesterday);
}

export function drawTheme(weights, rng = Math.random) {
    const entries = THEME_IDS.map((id) => {
        const raw = Number(weights?.[id] ?? 0);
        return [id, Number.isFinite(raw) && raw > 0 ? raw : 0];
    });
    const sum = entries.reduce((acc, [, w]) => acc + w, 0);
    if (sum === 0) {
        return THEME_IDS[Math.floor(rng() * THEME_IDS.length)];
    }
    const r = rng() * sum;
    let cumulative = 0;
    for (const [id, w] of entries) {
        cumulative += w;
        if (r < cumulative) return id;
    }
    return entries[entries.length - 1][0];
}

export function loadWeights() {
    if (!isBrowser()) return { ...DEFAULT_WEIGHTS };
    try {
        const raw = window.localStorage.getItem(STORAGE_KEY_WEIGHTS);
        if (!raw) return { ...DEFAULT_WEIGHTS };
        const parsed = JSON.parse(raw);
        const merged = { ...DEFAULT_WEIGHTS };
        for (const id of THEME_IDS) {
            const v = Number(parsed?.[id]);
            if (Number.isFinite(v) && v >= 0) merged[id] = v;
        }
        return merged;
    } catch {
        return { ...DEFAULT_WEIGHTS };
    }
}

export function saveWeights(weights) {
    if (!isBrowser()) return;
    const cleaned = {};
    for (const id of THEME_IDS) {
        const v = Number(weights?.[id]);
        cleaned[id] = Number.isFinite(v) && v >= 0 ? v : 0;
    }
    window.localStorage.setItem(STORAGE_KEY_WEIGHTS, JSON.stringify(cleaned));
}

export function loadDraws() {
    if (!isBrowser()) return [];
    try {
        const raw = window.localStorage.getItem(STORAGE_KEY_DRAWS);
        if (!raw) return [];
        const parsed = JSON.parse(raw);
        if (!Array.isArray(parsed)) return [];
        return parsed
            .filter(
                (d) =>
                    d &&
                    typeof d.night_date === 'string' &&
                    THEME_IDS.includes(d.theme),
            )
            .sort((a, b) => (a.night_date < b.night_date ? 1 : -1));
    } catch {
        return [];
    }
}

export function recordDraw(themeId, now = new Date()) {
    if (!THEME_IDS.includes(themeId)) {
        throw new Error(`Unknown theme id: ${themeId}`);
    }
    const nightDate = tonightDate(now);
    const cutoff = new Date(now);
    cutoff.setDate(cutoff.getDate() - RETENTION_DAYS);
    const cutoffString = toLocalDateString(cutoff);

    const existing = loadDraws();
    const pruned = existing.filter(
        (d) => d.night_date >= cutoffString && d.night_date !== nightDate,
    );
    const next = [
        { night_date: nightDate, theme: themeId, drawn_at: now.toISOString() },
        ...pruned,
    ].sort((a, b) => (a.night_date < b.night_date ? 1 : -1));

    if (isBrowser()) {
        window.localStorage.setItem(STORAGE_KEY_DRAWS, JSON.stringify(next));
    }
    return next;
}

export function lastNNightDates(n, now = new Date()) {
    const tonight = tonightDate(now);
    const base = new Date(`${tonight}T12:00:00`);
    const dates = [];
    for (let i = n - 1; i >= 0; i--) {
        const d = new Date(base);
        d.setDate(d.getDate() - i);
        dates.push(toLocalDateString(d));
    }
    return dates;
}
```

**Notes on key choices** (so future readers don't re-litigate them):
- `isBrowser()` guard is required because Next.js may run this file's top-level imports during server build / SSR, and `window` is undefined there. All localStorage entry points return safe defaults on the server.
- `loadWeights()` merges parsed values with `DEFAULT_WEIGHTS` per-key — protects against partial / stale persisted shapes if a key were ever renamed.
- `recordDraw`'s prune comparison `d.night_date >= cutoffString` works because `YYYY-MM-DD` strings sort lexicographically the same as chronologically.
- `recordDraw` filters out the existing entry for tonight before prepending — overwrite-on-collision semantics. The spec says "一天一抽不能重抽" so the UI gates this, but defensively replacing rather than duplicating keeps the array clean if the UI ever lets a duplicate slip through.
- `drawTheme` accepts an injectable `rng` so future debugging or seeded scenarios are possible without monkey-patching `Math.random`.
- `lastNNightDates` lives in this module (not in the page) because it shares the `tonightDate` cutoff logic — keeping date math centralized.

```jsx
'use client';

import { useEffect, useState } from 'react';
import { Dices } from 'lucide-react';
import { toast } from 'sonner';

import RecordPageLayout from '@/components/layout/RecordPageLayout';
import { THEMES_BY_ID } from '@/configs/nocturne-themes';
import {
    DEFAULT_WEIGHTS,
    drawTheme,
    loadDraws,
    loadWeights,
    recordDraw,
    tonightDate,
} from '@/lib/nocturne';

function PreNightPlaceholder({ now }) {
    const remaining = Math.max(0, 19 - now.getHours());
    return (
        <div className="my-12 text-center text-[#3f4a4e] opacity-70">
            <p className="text-lg font-bold">尚未到開放時間</p>
            <p className="mt-2 text-sm">
                7pm 後解鎖今晚的籤
                {remaining > 0 ? `（還有約 ${remaining} 小時）` : ''}
            </p>
        </div>
    );
}

function DrawButton({ onDraw }) {
    return (
        <button
            type="button"
            onClick={onDraw}
            className="my-8 w-full rounded-3xl bg-[#4a3f6b] py-16 text-2xl font-extrabold tracking-widest text-white shadow-lg transition active:scale-95"
        >
            <Dices className="mx-auto mb-3" size={44} strokeWidth={2.5} />
            抽今晚
        </button>
    );
}

function ThemeCard({ theme }) {
    return (
        <article
            className="my-6 rounded-2xl border-2 p-6"
            style={{
                borderColor: `${theme.color}66`,
                backgroundColor: `${theme.color}0d`,
            }}
        >
            <h2
                className="text-2xl font-extrabold tracking-wide"
                style={{ color: theme.color }}
            >
                {theme.name}
            </h2>

            {theme.entry && (
                <section
                    className="mt-5 rounded-xl p-4"
                    style={{ backgroundColor: theme.color, color: '#fff' }}
                >
                    <h3 className="text-xs font-bold uppercase tracking-widest opacity-80">
                        入場動作
                    </h3>
                    <p className="mt-2 text-base font-semibold leading-relaxed">
                        {theme.entry}
                    </p>
                </section>
            )}

            {theme.mainRule && (
                <section className="mt-5">
                    <h3 className="text-xs font-bold uppercase tracking-widest text-[#3f4a4e] opacity-60">
                        主場規則
                    </h3>
                    <p className="mt-2 text-sm leading-relaxed text-[#3f4a4e]">
                        {theme.mainRule}
                    </p>
                </section>
            )}

            {theme.rhythm && (
                <section className="mt-5">
                    <h3 className="text-xs font-bold uppercase tracking-widest text-[#3f4a4e] opacity-60">
                        節奏建議
                    </h3>
                    <p className="mt-2 text-sm leading-relaxed text-[#3f4a4e]">
                        {theme.rhythm}
                    </p>
                </section>
            )}
        </article>
    );
}

function FreeNightPoster({ theme }) {
    return (
        <article className="my-16 text-center">
            <h2 className="text-4xl font-extrabold tracking-[0.3em] text-[#3f4a4e]">
                {theme.name}
            </h2>
            <p className="mx-auto mt-10 max-w-md text-base leading-relaxed text-[#3f4a4e] opacity-80">
                {theme.mainRule}
            </p>
        </article>
    );
}

export default function NocturnePage() {
    const [mounted, setMounted] = useState(false);
    const [now, setNow] = useState(() => new Date());
    const [weights, setWeights] = useState(DEFAULT_WEIGHTS);
    const [draws, setDraws] = useState([]);

    useEffect(() => {
        setNow(new Date());
        setWeights(loadWeights());
        setDraws(loadDraws());
        setMounted(true);
    }, []);

    if (!mounted) {
        return (
            <RecordPageLayout title="夜籤">
                <div />
            </RecordPageLayout>
        );
    }

    const tonight = tonightDate(now);
    const tonightDraw = draws.find((d) => d.night_date === tonight) || null;
    const tonightTheme = tonightDraw ? THEMES_BY_ID[tonightDraw.theme] : null;

    const handleDraw = () => {
        const themeId = drawTheme(weights);
        const next = recordDraw(themeId, new Date());
        setDraws(next);
        toast.success('籤已抽出');
    };

    let mainArea;
    if (tonightTheme) {
        mainArea =
            tonightTheme.id === 'free' ? (
                <FreeNightPoster theme={tonightTheme} />
            ) : (
                <ThemeCard theme={tonightTheme} />
            );
    } else if (now.getHours() >= 19) {
        mainArea = <DrawButton onDraw={handleDraw} />;
    } else {
        mainArea = <PreNightPlaceholder now={now} />;
    }

    return <RecordPageLayout title="夜籤">{mainArea}</RecordPageLayout>;
}
```

**Notes on key choices:**
- `mounted` gate: first server render returns an empty stub, client hydrates, `useEffect` populates real state, then re-renders. This avoids a hydration mismatch warning between server-rendered "default state" and client-rendered "real state". The empty-stub flash is invisibly fast on real navigation.
- `now` is captured **once** in `useEffect` per design: spec accepts the 6:59→7:01 boundary edge case (page won't refresh mid-session). No `setInterval`.
- `recordDraw(themeId, new Date())` uses a **fresh** `new Date()` for the draw timestamp, distinct from the mount-time `now`. Otherwise a draw at 11pm would record `drawn_at` from when the page mounted.
- Tailwind arbitrary values (`text-[#4a3f6b]`, `tracking-[0.3em]`) follow the project pattern — see `app/page.jsx` for canonical use.
- Theme color is composed inline with hex alpha suffixes (`${color}66` = 40% opacity, `${color}0d` = 5% opacity) — same pattern as `app/page.jsx` does for the home menu border / hover. Tailwind 4 + arbitrary values would also work, but inline `style` is simpler when the color comes from data.
- `Dices` from lucide-react is the icon — semantically right for "draw a card / lottery", and it doesn't collide with `Moon` (used by sleep-tracker).
- `toast.success('籤已抽出')` — minimal feedback. The Toaster is already mounted by `RecordPageLayout`. Spec marked the toast optional; this is the chosen "small lightweight feedback" interpretation.

```jsx
function WeightSettings({ weights, onChange }) {
    const [open, setOpen] = useState(false);
    const sum = THEMES.reduce(
        (acc, t) => acc + (Number(weights[t.id]) || 0),
        0,
    );

    return (
        <section className="mt-auto border-t border-[#3f4a4e]/10 pt-4">
            <button
                type="button"
                onClick={() => setOpen((o) => !o)}
                className="flex w-full items-center justify-between text-sm font-bold uppercase tracking-widest text-[#3f4a4e] opacity-70"
            >
                <span className="flex items-center gap-2">
                    <Settings size={16} strokeWidth={2.5} />
                    權重設定
                </span>
                {open ? <ChevronUp size={16} /> : <ChevronDown size={16} />}
            </button>
            {open && (
                <div className="mt-4 space-y-3">
                    {THEMES.map((theme) => (
                        <div
                            key={theme.id}
                            className="flex items-center justify-between gap-3"
                        >
                            <span
                                className="flex-1 text-sm font-semibold"
                                style={{ color: theme.color }}
                            >
                                {theme.name}
                            </span>
                            <input
                                type="number"
                                min={0}
                                step={1}
                                value={weights[theme.id] ?? 0}
                                onChange={(e) => {
                                    const v = Number(e.target.value);
                                    onChange(
                                        theme.id,
                                        Number.isFinite(v) && v >= 0 ? v : 0,
                                    );
                                }}
                                className="w-20 rounded border border-[#3f4a4e]/20 bg-white px-2 py-1 text-right text-sm text-[#3f4a4e] focus:border-[#4a3f6b] focus:outline-none"
                            />
                        </div>
                    ))}
                    <div className="flex justify-between border-t border-[#3f4a4e]/10 pt-2 text-xs text-[#3f4a4e] opacity-60">
                        <span>總和</span>
                        <span>{sum}</span>
                    </div>
                </div>
            )}
        </section>
    );
}
```

**Notes on key choices:**
- `mt-auto` on the outer `<section>`: thanks to the parent `RecordPageLayout`'s `flex flex-col grow` on `<main>`, this pushes the settings block to the bottom of the viewport when there's spare vertical space. On a tall screen the settings sit at the page bottom; on a short / scrolling page they just follow the history strip naturally.
- Number input is the simplest form factor that fits any screen width and gives the user immediate keyboard control. +/- buttons or sliders were spec open questions; this is the chosen baseline. Easy to swap later.
- `min={0} step={1}` + the `onChange` clamp protects against negatives and NaN. The spec says "client 端 input 自然限制即可，UI 不出現負號" — `min={0}` enforces this without explicit error UI.
- Sum is shown but no validation enforces total = 100. This matches spec line 36 AC.
- Theme name is colored with its theme color — gives the settings list a quick visual key matching the cards.
